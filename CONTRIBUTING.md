# Contributing to WebCompilerStudio

## How to contribute

1. **Fork** the repository and clone your fork.
2. **Create a branch** for your change (`git checkout -b my-feature`).
3. **Write code** following the existing style (TypeScript, Prettier, ESLint).
4. **Add tests** for new functionality.
5. **Run the CI** locally (`npm test`).
6. **Commit** with a clear message and push to your fork.
7. **Open a Pull Request** against the `main` branch.

## Pull Request checklist
- [ ] Code follows project linting rules.
- [ ] Tests added / existing tests pass.
- [ ] Documentation updated if needed.
- [ ] PR description includes purpose, changes, and any relevant issue references.
